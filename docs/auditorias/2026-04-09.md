## 2. Gaps detectados (top 8 por impacto)

### 🔴 CRÍTICO — Gap 1: No existe `.claudeignore`

**Problema:** Cada vez que Claude explora el workspace para encontrar contexto, lee carpetas como:
- `Proyectos Internos/kitcha-landing/.next/` — cientos de archivos JS minificados
- `clientes/constanza-nutricion/Nutrisco/node_modules/` — miles de archivos
- `.git/objects/` en todos los repos (white-cassini, nutrisco, etc.)

Esto consume tokens masivamente y puede llegar a saturar el context window sin aportar información útil.

**Solución:** Crear `creactive-studio/.claudeignore` con:
```
.next/
node_modules/
.git/
__pycache__/
*.pyc
.DS_Store
*.sst
*.meta
.tsbuildinfo
```

**Impacto estimado:** Reducción del 60–80% en tokens consumidos al explorar proyectos con builds.

---

### 🔴 CRÍTICO — Gap 2: Path hardcodeado en `sal-propuesta`

**Problema:** La Fase 3 del comando `sal-propuesta` tiene:
```
/Users/oz/Documents/CAS-CEO/creactive-studio/plantillas/propuesta-comercial/template.html
```

Este path absoluto rompe el skill si:
- Se trabaja desde una sesión diferente
- El workspace cambia de ubicación
- Se trabaja desde Cowork vs. Claude Code

```

```

## 3. Oportunidades de optimización de tokens

### 3.1 El problema raíz: contexto que carga todo el tiempo

El `CLAUDE.md` raíz tiene 564 líneas y es cargado automáticamente en cada sesión. Incluye:
- Tabla completa de stack tecnológico (muchos datos raramente consultados)
- Árbol de directorios del workspace (sección 11) — redundante porque Claude puede verlo directamente
- Perfil personal de Oscar — útil para propuestas pero no para desarrollo

**Oportunidad:** Sin reducir la calidad del contexto, se puede ahorrar ~30% de tokens en el CLAUDE.md raíz eliminando la sección 11 (árbol de archivos) que está desactualizada de todas formas, y comprimiendo la tabla de stack en sección 3.

### 3.2 Nutrisco: 4 niveles de CLAUDE.md anidados

El proyecto Nutrisco tiene CLAUDE.md en:
- `clientes/constanza-nutricion/CLAUDE.md` (543 líneas — el más completo)
- `clientes/constanza-nutricion/Nutrisco/CLAUDE.md`
- `clientes/constanza-nutricion/Nutrisco/app/CLAUDE.md`
- `clientes/constanza-nutricion/Nutrisco/app/src/lib/supabase/CLAUDE.md`

Cuando Claude trabaja dentro de `Nutrisco/app/`, carga los 4 archivos encadenados. Si hay información redundante entre ellos, es tokens malgastados.

**Acción:** Revisar si los CLAUDE.md de niveles más profundos tienen info que no está en el de nivel superior, y eliminar redundancias.

### 3.3 Content engine: 3 archivos de contexto cargados por cada comando de contenido

`/cnt-carrusel`, `/cnt-content-plan`, `/cnt-video-brief` y `/cnt-reel-guion` leen cada uno:
- `content-engine/oscar/voz.md`
- `content-engine/oscar/pilares.md`
- `content-engine/oscar/hormozi-patterns.md` (solo cnt-content-plan)

Si ejecutas 2-3 comandos de contenido en una sesión, esos archivos se leen múltiples veces.

**Oportunidad:** Si hay una sesión de creación de contenido, cargar los archivos de contexto de Oscar manualmente al inicio de la conversación en lugar de que cada comando los re-lea.

### 3.4 `.next/` y `node_modules/` sin `.claudeignore`

Como se describe en Gap 1, este es el mayor desperdicio de tokens del workspace. Tiene solución inmediata.
